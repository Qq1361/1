# M4-B2 每日经营报告页面

## 已完成范围

路由为 `/reports/daily`，导航名称为“每日经营报告”。页面只请求 `GET /api/reports/daily-business`，支持通过 `date=YYYY-MM-DD` 查询指定报告日期；`timezone` 固定为 `Asia/Shanghai`。

初次进入不传 `date`，由 API 使用北京时间昨天。用户选择日期后，页面将日期同步到 URL，浏览器刷新仍会使用同一日期。“查看昨日”会移除日期参数，重新使用 API 默认日期。

## 展示边界

- 所选日期经营结果：销售、采购和行情事件属于所选北京时间日期。
- 当前库存与资产、当前待办、当前风险：均为 `generatedAt` 时刻快照，不代表所选历史日期当天状态。
- 销售净到账、售后净利润、资产成本、待办数量和风险阈值完全由 M4-B1 服务端聚合决定；页面不重新计算。
- 行情摘要明确为人工录入，不代表系统自动读取得物或 95 分数据。

## 只读与非目标

页面没有 Prisma 访问、没有 ownerId 输入、没有采购/销售/库存/退款写入，也不保存日报。M4-B3 已增加“发送到飞书”：用户须先确认报告日期，浏览器只调用本系统的发送 API，不显示、保存或提交 Webhook/Secret。发送期间固定日期并禁用重复提交；失败不会清空日报，成功不会修改日报内容。

手动发送复用 M4-B4 的发送记录和同日幂等状态；失败时按服务端错误类型显示可重试或不可重试结果。M4-A7、M4-A8 继续暂停。

## 错误与移动端

400 参数错误、500 生成失败和网络失败均显示中文错误与重新加载入口。新请求开始后隐藏旧报告，避免把旧数据伪装成新日期结果。页面采用卡片网格和纵向待办/风险卡片，手机端不要求横向滚动查看核心指标。
