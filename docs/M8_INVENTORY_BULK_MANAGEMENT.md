# M8 库存列表全选与批量维护

## 选择范围

- 库存列表使用一套选择状态，支持单件、全选当前页、取消当前页和跨分页累计选择。
- “全选当前筛选结果”由服务端按当前搜索、筛选和排序返回完整 ID 集；首版上限为 200 件，超过上限返回 `INVENTORY_BULK_TOO_MANY`，用户必须缩小范围。
- 搜索、筛选或排序变化会清空选择。列表刷新时会复核已选 ID，仅移除已不存在或不再属于当前 owner 的 ID；仍有效的跨页选择会保留。
- 工具栏统计基于全部已选库存，而非当前页：商品/SKU、仓库和库存状态均覆盖跨页选择。

## 可维护范围和操作

只有 `STOCKED` 且 `OWNED` 的库存可进行本阶段的仓位、成色、计划出售方式和保质期维护。每项操作有独立入口及独立弹窗，固定操作类型后才可填写对应字段：

- 调整仓位：必须同时选择当前 owner 的启用仓库和该仓库的启用库位；不清空历史自由文本库位、不创建调拨单、不变更库存状态。
- 设置成色：仅接受 `InventoryCondition` 枚举；跨商品/SKU 必须明确确认，不反向改写验货事实。
- 设置计划出售方式：仅更新现有 `saleMode`（包括 `NONE`）；不会创建销售单、平台寄售，也不会写入 `PLATFORM_LISTED` 或 `SOLD`。
- 修正保质期：生产日期、保质期月数、到期日分别支持保持、设置、清空；到期日还支持按日历月截断规则自动计算。日期只接受 `YYYY-MM-DD`，不自动纠正非法日期，到期日不得早于生产日期，且必须填写修正原因；不会修改采购商品快照。
- 批量设置 SKU/色号继续沿用原有独立规则：可选择覆盖已有 SKU、跨商品确认以及受控的历史售出档案修正。该历史权限不会扩散到其他批量操作。

## 预览、提交和锁定

所有新增维护操作均为“选择 → 填写 → 服务端预览 → 用户确认 → 原子提交”。预览不写库，返回每件变更前后、可变更数、阻断项和原因，以及基于当前快照的 `selectionFingerprint`。提交在 Serializable 事务中重新读取并比对指纹，任何数据变化、重复 ID、超出 200 件、跨 owner 或任一锁定项都会拒绝整批，不支持部分成功。

服务端会阻断：非 `STOCKED`（含 `SOLD`）状态、非自有库存、销售明细关联、平台寄售/退回关联、采购售后关联和销售售后关联。操作不会变更 `itemStatus` 或单位成本，也不会创建销售、售后、退款、平台寄售或物流记录。

## 审计日志

每个成功变更的库存都会在同一事务内写入 `InventoryItemActionLog`。日志包含 owner、库存 ID、同批 `batchId`、操作类型、变更前后 JSON 快照、可选原因和创建时间；任意失败会与库存修改一并回滚。日志模型的外键采用 `RESTRICT`，不保存凭证、密码或无关隐私。

## 交互结果

成功后关闭当前操作弹窗、清空选择、刷新列表，并保持当前搜索和筛选条件，同时显示实际更新数量。失败时保留输入与选择，预览会展示具体受阻库存及原因；发生指纹冲突时要求重新预览。

## 验证

```powershell
pnpm verify:m8-inventory-bulk-management
```

该脚本使用独立 owner 和精确清理的临时夹具，覆盖选择契约、上限、目标仓位校验、元数据更新、保质期日期规则、锁定、事务回滚、并发指纹和审计日志。
