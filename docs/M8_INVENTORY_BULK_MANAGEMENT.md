# M8 库存列表全选与批量维护

## 选择范围

- 库存列表使用一套选择状态，支持单件、全选当前页、取消当前页和跨分页累计选择。
- “全选当前筛选结果”由服务端按当前搜索、筛选和排序返回完整 ID 集；首版上限为 200 件，超过上限返回 `INVENTORY_BULK_TOO_MANY`，用户必须缩小范围。
- 搜索、筛选或排序变化会清空选择。列表刷新时会复核已选 ID，仅移除已不存在或不再属于当前 owner 的 ID；仍有效的跨页选择会保留。
- 工具栏统计基于全部已选库存，而非当前页：商品/SKU、仓库和库存状态均覆盖跨页选择。

## 可维护范围和操作

只有 `STOCKED` 且 `OWNED` 的库存可进行本阶段的仓位、成色、计划出售方式和保质期维护。每项操作有独立入口及独立弹窗，固定操作类型后才可填写对应字段：

- 调整仓位：必须显式选择 `locationMode`。默认 `MANUAL` 使用当前 owner 的启用仓库和手动文本，保存时清空 `storageLocationId`；可选 `STANDARD` 使用启用仓库及其启用标准库位，保存时清空新的 `storageLocation`。两种模式互斥，不创建调拨单，也不变更库存状态。A/B/C/D 仓由用户创建，不写死在代码中。
- 设置成色：仅接受 `InventoryCondition` 枚举；跨商品/SKU 必须明确确认，不反向改写验货事实。
- 设置计划出售方式：仅更新现有 `saleMode`（包括 `NONE`）；不会创建销售单、平台寄售，也不会写入 `PLATFORM_LISTED` 或 `SOLD`。
- 修正保质期：生产日期、保质期月数、到期日分别支持保持、设置、清空；到期日还支持按日历月截断规则自动计算。日期只接受 `YYYY-MM-DD`，不自动纠正非法日期，到期日不得早于生产日期，且必须填写修正原因；不会修改采购商品快照。
- 批量设置 SKU/色号使用独立的“服务端逐件预览 → 指纹确认”流程：预览逐件显示旧 SKU、新 SKU 与处理结果，默认只补空 SKU，已有 SKU 保留；跨商品必须明确确认。已售出库存档案默认排除，只有 SKU 纠错可显式包含历史档案，该历史权限不会扩散到其他批量操作。

## 预览、提交和锁定

仓位、成色、计划出售方式和保质期维护均为“选择 → 填写 → 服务端预览 → 用户确认 → 原子提交”。预览不写库，返回每件变更前后、可变更数、阻断项和原因，以及基于当前快照的 `selectionFingerprint`。提交在 Serializable 事务中重新读取并比对指纹，任何数据变化、重复 ID、超出 200 件、跨 owner 或任一锁定项都会拒绝整批，不支持部分成功。

SKU 纠错同样必须先预览，再携带 `selectionFingerprint` 确认；确认阶段对每件待更新库存比对 `updatedAt`，任一并发变化都会回滚本次待更新集合。SKU 预览与确认只修改 `InventoryItem.skuText` 和对应审计日志，不回写任何 `SaleLine.skuSnapshot`，不写库存状态、成本、销售、寄送或售后事实。

服务端会阻断：非 `STOCKED`（含 `SOLD`）状态、非自有库存、销售明细关联、平台寄售/退回关联、采购售后关联和销售售后关联。操作不会变更 `itemStatus` 或单位成本，也不会创建销售、售后、退款、平台寄售或物流记录。

## 审计日志

每个成功变更的库存都会在同一事务内写入 `InventoryItemActionLog`。位置快照包含仓库 ID/名称、`locationMode`、标准库位 ID/名称和自由文本；因此预览与历史日志不会只显示 ID。日志还包含 owner、库存 ID、同批 `batchId`、操作类型、可选原因和创建时间；任意失败会与库存修改一并回滚。日志模型的外键采用 `RESTRICT`，不保存凭证、密码或无关隐私。

## 交互结果

成功后关闭当前操作弹窗、清空选择、刷新列表，并保持当前搜索和筛选条件，同时显示实际更新数量。失败时保留输入与选择，预览会展示具体受阻库存及原因；发生指纹冲突时要求重新预览。

## 验证

```powershell
pnpm verify:m8-inventory-bulk-management
pnpm verify:m8-inventory-bulk-browser
pnpm verify:m8-manual-storage-location
```

服务验证脚本使用独立 owner 和精确清理的临时夹具，覆盖选择契约、上限、手动/标准目标仓位校验、元数据更新、SKU 预览与并发指纹、保质期日期规则、锁定、事务回滚和审计日志。浏览器验证脚本使用独立端口、访问会话与唯一前缀夹具，真实执行批量验货手动库位、跨页选择和五类批量操作链路，并在 1440px 与 390px 验收后精确清理夹具、浏览器和临时服务。
