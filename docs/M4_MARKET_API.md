# M4-A3 Market API Contract

M4-A3 provides APP_PASSWORD-protected, owner-scoped APIs for market items and manual quotes. Money is returned as a two-decimal string. Dates are ISO 8601 strings or null. Clients cannot submit ownerId, normalization values, quote sources, or quote lifecycle timestamps.

## Routes

| Method | Path | Purpose |
|---|---|---|
| GET / POST | `/api/market/items` | List or create market items |
| GET / PATCH | `/api/market/items/[marketItemId]` | Read or update a market item |
| POST | `/api/market/items/[marketItemId]/deactivate` | Deactivate a market item |
| POST | `/api/market/items/[marketItemId]/reactivate` | Reactivate a market item |
| GET / POST | `/api/market/items/[marketItemId]/quotes` | List quote history or create a manual quote |
| POST | `/api/market/quotes/[quoteId]/confirm` | Confirm a quote |
| POST | `/api/market/quotes/[quoteId]/invalidate` | Invalidate a quote |
| POST | `/api/market/quotes/[quoteId]/replace` | Atomically invalidate a quote and create its replacement |

`GET /api/market/items` accepts `keyword`, `platform`, `quoteType`, `active`, `lifecycleStatus`, `hasCurrentQuote`, `page`, and `pageSize`. Its response includes `items`, `pagination`, and `appliedFilters`.

Quote history accepts `platform`, `quoteType`, `lifecycleStatus`, `dateFrom`, `dateTo`, `page`, and `pageSize`. UI remains out of scope; these APIs are the fact source for a later UI phase.

## Write Boundaries

- MarketItem normalization is generated by the Service. Potential duplicates are warnings only and are never merged automatically.
- A created quote is always `MANUAL`. `confirmImmediately` only asks the Service to set a server-side confirmation time.
- Quote amount, platform, type, recorded time, and market-item ownership are never overwritten. Corrections must call `replace`, which invalidates the original and creates a new quote in one transaction.
- Repeating quote confirmation and same-reason invalidation are idempotent.
- Serializable replacement write conflicts return `409 MARKET_QUOTE_CORRECTION_CONFLICT`. They do not create two replacement quotes.
- Routes only delegate to Market Service and Query. They do not directly write Prisma, purchase, inventory, sales, after-sales, or `SOLD`.

## Error Contract

```json
{
  "error": {
    "code": "MARKET_ITEM_NOT_FOUND",
    "message": "User-safe message",
    "fieldErrors": {}
  }
}
```

- Invalid parameters, dates, money formats, or unknown fields: `400`.
- Missing or cross-owner resources: `404`, without resource-existence disclosure.
- Inactive item writes, invalidated quote confirmation, invalidation conflicts, and replacement concurrency conflicts: `409`.
- Unexpected errors: `500 INTERNAL_ERROR`, without a stack trace or database details.

## Out of Scope

M4-A3 adds no UI, decision calculator, estimated profit, maximum purchase price, platform recommendation, charts, notifications, automated collection, or inventory/purchase/sales writes.

## M4-A4 UI 使用约定

`/market` 和 `/market/[marketItemId]` 是 M4-A4 的唯一行情工作台路由。客户端只调用本文定义的 `/api/market/*` 接口：不提交 `ownerId`、`normalizedName`、`normalizedSku` 或 Service 控制的 Quote 生命周期时间字段；不直接写入数据库。

## M4-A6 采购试算

`POST /api/market/items/[marketItemId]/purchase-decision` 为只读即时试算。仅使用当前有效 `EXPECTED_INCOME`，不使用展示价格或人工参考价，不保存决策，也不创建采购订单。详见 [采购试算 API](./M4_MARKET_PURCHASE_DECISION_API.md)。

详情页依赖 `currentQuotesByPlatform`、`availabilityReason` 和 `availableActions` 渲染当前行情与操作入口。列表页的筛选参数为 `keyword`、`lifecycleStatus`、`hasCurrentQuote`、`platform`、`page`、`pageSize`，并保留在 URL 中。
