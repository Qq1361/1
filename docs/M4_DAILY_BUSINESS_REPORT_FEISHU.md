# M4-B3 飞书日报手动发送

## 范围

M4-B3 仅支持飞书群自定义机器人 Webhook 的手动日报发送。调用 `POST /api/reports/daily-business/send-feishu` 时，服务端复用 M4-B1 的日报聚合，再生成不含明细样本的文本摘要。

环境变量只在服务端读取：

- `FEISHU_DAILY_REPORT_WEBHOOK_URL`：必填。
- `FEISHU_DAILY_REPORT_SECRET`：可选；配置后按飞书 V2 自定义机器人签名协议发送 `timestamp` 与 `sign`。

Webhook、签名密钥、签名原文和完整飞书响应不会返回浏览器、写入数据库或打印到业务日志。生产环境仅接受 HTTPS Webhook；非生产环境只允许本地回环 HTTP，以支持隔离 mock 验证。

## 消息边界

摘要包含所选日期的销售结果、生成时刻的库存资产快照、P0/P1 待办、重要风险和人工行情说明。它不包含客户资料、地址、聊天内容、数据库 ID、订单明细或售后说明。待办和风险只发送分类数量；过长内容按优先级缩减并保留核心金额。

`EXPECTED_INCOME`、实际到账、退款和利润仍全部来自既有日报 DTO。格式化层不查询数据库，也不重新计算金额。

## 操作与错误

日报页面的“发送到飞书”会先显示确认弹窗，并固定当次日期。成功后仅显示 Toast，不改变当前报告。配置缺失、超时、网络问题、飞书 HTTP/业务拒绝和非 JSON 响应均使用稳定中文错误，不回显飞书响应。

M4-B3 的手动发送现由 M4-B4 的 Delivery Coordinator 统一处理；同一 owner、日期和飞书渠道不会重复发送已成功记录，网络/超时/5xx 失败可按规则重试。
# M4-B4 发送记录

普通手动发送和 CLI 均通过 Delivery Coordinator 处理同日幂等。Webhook 与 Secret 仍只从服务端环境变量读取；发送记录不保存配置、正文或外部原始响应。
